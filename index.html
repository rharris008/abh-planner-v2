Data freshness

Who it's for: Pureau fulfilment + sales teams.

Single question this answers: over the next 14 days, what will the stock-on-hand look like at end of each day, and which dispatches are at risk?

Two views you actually need:

  • Closing stock heatmap — the at-a-glance view. Each cell = end-of-day stock per SKU per day. Red = shortfall, can't ship that day in full.
  • Overdue dispatches — orders that should have shipped already. These ship first.

Audit tab: 100% line-by-line. Every forecast row, every SO line, every public holiday, every production assumption. If a number is in the heatmap, you can trace it from the audit tab back to the NetSuite source.

Production assumption: 10L Cask = 11,520 units/day weekday (120 pal × 96 UPP, target across HMPS + Line A). 5L Cask = 1,884 units/day weekday (Line A alternates with 10L during Line B outage 13/04-13/06). 2L = 1,670; 600ml 12pk = 2,948; 600ml 6pk = 4,120 — W14-W17 NS Build actual averages. Saturday = half weekday rate for cask SKUs only; bottle lines weekday-only. Sundays + public holidays = 0.

Demand sources: WW + Coles forecasts use order_date per row (their actual dispatch date). Metcash spreads across workdays (no order_date). Confirmed open SOs sit on their ship_date. Per-day = MAX(forecast on that day, SOs on that day) to avoid double-count. Anything falling on PH/weekend shifts to next workday.
